# Yaru Version Release and Deployment Guide

Last updated: 2026-03-04

This document explains how to release new Yaru versions and publish update assets via GitHub Release.

## 1. Scripts and Paths

Release scripts:

- `Monorepo/scripts/deploy.py`
- `Monorepo/scripts/deploy_gui.py`

Key paths:

- Flutter app: `Monorepo/apps/Flutter`
- Output directory: `Monorepo/dist`
- Update manifest: `latest.json` (generated by script, uploaded as GitHub release asset)

## 2. Environment Setup

### 2.1 Python Dependencies

```bash
pip install pyyaml
```

### 2.2 Required Environment Variables

```bash
export GH_TOKEN="<github-token-with-release-write>"
export UPDATE_GITHUB_REPO="ProjectYaru/Yaru-release"
```

## 3. Pre-Release Checklist

1. Confirm version number in `Monorepo/apps/Flutter/pubspec.yaml` (`version: x.y.z+build`).
2. Update changelog in `Monorepo/apps/Flutter/CHANGELOG.md`.
3. Confirm code is buildable:

```bash
cd Monorepo/apps/Flutter
flutter pub get
flutter test
```

## 4. Deployment Commands

Run in `Monorepo` directory:

```bash
# GUI release (one-click GitHub Release publish)
python scripts/deploy_gui.py

# Build/package only (no publish)
python scripts/deploy.py --build-only

# Build and publish for current platform
python scripts/deploy.py

# Specify platform
python scripts/deploy.py --platform windows
python scripts/deploy.py --platform linux
python scripts/deploy.py --platform macos
python scripts/deploy.py --platform android
```

## 5. GUI Release Notes

`scripts/deploy_gui.py` supports:

- Select platform artifact path and automatically calculate `SHA-256` / file size.
- Maintain Chinese/English changelog entries (`feature/fix/improvement/breaking`).
- Configure force update and `min_supported_version`.
- Pull latest online `latest.json` from GitHub release and merge per-platform assets.
- One-click create/update GitHub release and upload:
  - platform installers
  - `latest.json`
  - `history-<version>.json`

## 6. Artifacts and Manifest

The release process generates:

1. Platform artifacts in `Monorepo/dist/<version>/<platform>/`
2. `Monorepo/dist/<version>/latest.json`
3. GitHub release assets containing installers and `latest.json`

`latest.json` download URLs point directly to GitHub release assets.

## 7. Release Acceptance (Definition of Done)

1. Target platform builds successfully and artifacts exist in `Monorepo/dist/<version>/<platform>/`.
2. GitHub latest release includes `latest.json` and platform installers.
3. `https://update.asaka.moe/latest.json` is reachable and returns the latest manifest.
4. Client update check reads the new version and can complete update flow.

## 8. Common Issues

- `gh` authentication errors: ensure `GH_TOKEN` has release write permissions.
- Upload failed: verify repository name format (`owner/repo`) and token scope.
- Manifest mismatch: re-run packaging and verify hash/size values in `latest.json`.
